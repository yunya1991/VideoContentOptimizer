## 视频格式与规格限制

### 支持哪些视频格式？

| 格式 | 支持 | 说明 |
|---|---|---|
| `.mp4` | ✅ | 最推荐，兼容性最好 |
| `.mov` | ✅ | Apple 设备录制常见格式 |
| `.avi` | ✅ | 老格式，文件较大 |
| `.mkv` | ✅ | 高清视频常见格式 |
| `.wmv` / `.flv` / `.webm` | ❌ | 暂不支持，可先用 FFmpeg 转为 mp4 |

**转换命令：**
```bash
ffmpeg -i input.wmv output.mp4
```

### 视频文件可以多大？

默认上限 **500 MB**。如果你的视频更大，可在 `.env` 中调整：

```bash
MAX_VIDEO_SIZE_MB=1000   # 改为 1 GB
```

### 视频可以多长？

没有硬性时长限制，但请注意：

| 视频时长 | 转录耗时（CPU + base 模型） | 建议 |
|---|---|---|
| ≤ 5 分钟 | < 1 分钟 | 正常使用 |
| 5–30 分钟 | 1–8 分钟 | 正常，稍等 |
| 30–60 分钟 | 8–15 分钟 | 建议先剪辑到关键片段 |
| > 60 分钟 | 15 分钟以上 | 强烈建议分段处理 |

**加速方法：**
```bash
WHISPER_DEVICE=cuda           # 有 NVIDIA GPU 时开启，速度快 5-10 倍
WHISPER_MODEL_SIZE=tiny       # 速度更快，精度略低
WHISPER_MODEL_SIZE=small      # 平衡速度和精度
```

### 视频分辨率有要求吗？

没有。本工具处理的是音频轨和文字内容，不受分辨率影响。即使是低质量视频，只要有清晰的语音，分析结果就准确。

## 启动与配置问题

### 启动时报错"生产环境禁止使用默认弱密钥"

**这是什么意思？**
系统检测到你没有修改默认的安全密钥，为了防止部署到公网后被攻击，拒绝启动。

**怎么解决？**

第一步，生成两个随机密钥：
```bash
python -c "import secrets; print(secrets.token_hex(32))"
# 运行两次，得到两个不同的密钥
```

第二步，把密钥填入 `.env`：
```bash
SECRET_KEY=你刚才生成的第一个密钥
JWT_SECRET=你刚才生成的第二个密钥
```

第三步，重启服务。

> 如果只是本地开发，也可以临时设置 `API_DEBUG=true` 跳过此检查，但不要在公网服务器上这样做。

### API 服务启动了，但 Web UI 显示"API 服务未连接"

按顺序检查：

1. **确认 API 是否真的在跑：** 浏览器访问 `http://localhost:8080/health`，看到 `{"status": "ok"}` 即正常
2. **端口冲突：** 如果 8080 被占用，改用其他端口：
   ```bash
   uvicorn app.main:app --port 8090
   # 同时设置 API_URL 环境变量
   API_URL=http://localhost:8090 streamlit run webui/main.py
   ```
3. **防火墙：** Linux 服务器上确认 8080 端口已开放

### `.env` 文件改了，但好像没生效

改 `.env` 后必须重启服务，配置不会热加载：
```bash
# 停止现有服务（Ctrl+C 或 kill 进程）
# 重新启动
uvicorn app.main:app --host 0.0.0.0 --port 8080 --reload
```

### 提示 `ModuleNotFoundError: No module named 'anthropic'`

安装 Anthropic SDK：
```bash
pip install anthropic
```

这个包在 `requirements.txt` 里是可选的，使用 Claude 时才需要。

## LLM / AI 相关问题

### 必须要有 API Key 才能用吗？

**视频分析和优化**必须有 LLM API Key（用于意图识别、文案优化等 AI 功能）。

**不需要** API Key 的功能：
- 视频元数据提取（时长、分辨率、帧率）
- edge TTS 配音（免费，联网即可）
- 字幕烧录（本地 FFmpeg 处理）

**免费额度建议：**
- 国内用户推荐 [SiliconFlow](https://siliconflow.cn)，每日有免费额度
- 性价比最高：DeepSeek，按量计费，价格极低

### API 调用失败，提示 429 错误

**这是什么意思？** 请求太频繁，超过了 API 提供商的速率限制。

**解决方法：**
1. 减少并发数：在 `.env` 中设置 `PARALLEL_WORKERS=1`
2. 稍等 1-2 分钟后重试
3. 批量处理时设置 `BATCH_SIZE=2`

### LLM 返回的内容是乱码或者空白

可能原因和解决方法：

| 原因 | 检查方法 | 解决 |
|---|---|---|
| API Key 余额不足 | 登录对应平台查看余额 | 充值或换一个 provider |
| 模型名称填错 | 查看 `logs/app.log` 中的错误详情 | 参照 SKILL.md 中的模型名称列表 |
| 网络超时 | `logs/app.log` 中有 `timeout` 字样 | 检查网络，或增大超时时间 |

### 如何切换到 Claude / GPT-4 等其他模型？

在 `.env` 中修改（改完重启服务）：

```bash
# 切换到 Claude
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=sk-ant-xxx
LLM_MODEL=claude-sonnet-4-6

# 或者用模型前缀一行搞定（不用改 LLM_PROVIDER）
LLM_MODEL=anthropic:claude-opus-4-7
```

## 智能优化：询问/执行模式

### 推荐卡片和参数表单，用哪个更好？

没有绝对答案，看你的使用习惯：

- **推荐卡片**：适合不确定往哪个方向优化时，让 AI 给几个选项。代价是多消耗约 500–1000 tokens（相当于多问 AI 一次小问题）。
- **参数表单**：适合已经清楚自己要什么的用户，比如"我就是要做抖音爆款，活泼风格"。没有额外 token 消耗。

**经验之谈**：新内容/新品类用推荐卡片，固定套路的内容用参数表单更高效。

### 为什么推荐卡片生成失败了？

最常见的原因（按概率排序）：

1. **API 服务没启动** → 浏览器访问 `http://localhost:8080/health` 确认
2. **LLM_API_KEY 没配置** → 检查 `.env` 文件
3. **LLM API 余额不足** → 查看对应平台余额
4. **网络问题** → 查看 `logs/app.log` 中的具体错误

> 推荐卡片失败后系统会自动降级为 3 个静态默认档案（爆款流量型 / 知识干货型 / 品牌调性型），不会报错，可以继续使用。

### 选了推荐方向，但优化结果和预期不太一样？

当前版本中，参数选择（平台/目标/调性）主要用于进化引擎学习你的偏好，优化引擎本身还未完全接入这些参数。这是已知的待完善项，后续版本会让优化结果更贴合你的选择。

### 批量执行模式下，参数预设收起来了，用的是什么参数？

收起折叠面板时，使用上一次执行的参数；如果从未执行过，使用默认值：

| 参数 | 默认值 |
|---|---|
| 目标平台 | 抖音（douyin） |
| 优化目标 | 互动提升（engagement） |
| 内容调性 | 活泼有趣（energetic） |
| 标题变体数 | 5 个 |

## TTS 配音问题

### 免费的 edge TTS 有什么限制？

- 需要联网（调用微软服务）
- 单次最多约 3000 字，超出自动分段
- 速度中等，适合大多数使用场景
- 音色选择丰富，支持中文多地区口音

### TTS 生成的音频文件是空的

**原因 1：网络不通（最常见）**
edge TTS 需要访问微软服务器，检查网络或挂代理。

**临时解决方案：** 换用不需要联网的方案（目前暂无完全离线 TTS，建议换 SiliconFlow）：
```bash
TTS_VOICE_NAME=siliconflow:anna
SILICONFLOW_API_KEY=sk-xxx
```

**原因 2：临时目录没有写入权限**
```bash
ls -la /tmp/video_optimizer/   # 检查目录是否存在和有写入权限
mkdir -p /tmp/video_optimizer  # 手动创建
```

### 如何试听不同音色？

目前没有内置试听功能，可以用命令行快速预览：
```bash
pip install edge-tts
edge-tts --voice zh-CN-XiaoxiaoNeural --text "你好，这是测试。" --write-media test.mp3
```

常用中文音色：
- `zh-CN-XiaoxiaoNeural` — 女声，活泼（最常用）
- `zh-CN-YunxiNeural` — 男声，通用
- `zh-CN-XiaoyiNeural` — 女声，温柔
- `zh-CN-YunjianNeural` — 男声，激情（适合运动/励志内容）

## 发布与平台问题

### 抖音/小红书/微信视频号支持发布吗？

| 平台 | 发布支持 | 说明 |
|---|---|---|
| 抖音（douyin） | ✅（映射到 TikTok） | 通过 upload-post.com 发布，需要注册账号 |
| TikTok | ✅ | 直接支持 |
| YouTube | ✅ | 直接支持 |
| Instagram | ✅ | 直接支持 |
| 小红书 | ❌ | upload-post.com 暂不支持，跳过 |
| 微信视频号 | ❌ | upload-post.com 暂不支持，跳过 |

> 小红书和微信视频号目前需要手动发布，本工具可以帮你生成优化好的标题和文案，复制粘贴即可。

### 发布接口返回 503，提示"发布功能未启用"

在 `.env` 中开启：
```bash
UPLOAD_POST_ENABLED=true
UPLOAD_POST_API_KEY=你的API Key   # 去 upload-post.com 注册获取
UPLOAD_POST_USERNAME=你的用户名
```

改完重启服务。

### 发布到 TikTok 后，视频审核被拒

这是平台内容审核问题，与本工具无关。建议检查视频内容是否符合目标平台社区准则。

## 性能与速度问题

### 整体处理流程大概要多久？

| 操作 | 典型耗时 | 主要瓶颈 |
|---|---|---|
| 视频分析（5 分钟视频） | 1–3 分钟 | Whisper 转录 |
| 生成推荐卡片 | 5–15 秒 | LLM API 响应 |
| 文案优化 | 5–15 秒 | LLM API 响应 |
| 标题生成（5 个） | 5–15 秒 | LLM API 响应 |
| TTS 配音（1000 字） | 5–20 秒 | edge TTS 网络 |

### 批量处理多段文案，速度很慢

批量执行模式是串行逐段处理的，每段的速度取决于 LLM API 响应速度。

**加速建议：**
1. 选择响应快的 LLM（DeepSeek 国内响应较快）
2. 减少标题变体数（`num_variants=3` 代替默认的 5）
3. 只开启需要的优化类型（例如只选"标题生成"）

## 进化引擎与用户画像

### 什么是"进化引擎"和"Soul 用户画像"？

**进化引擎**：记录你每次使用的操作（成功/纠正），自动从中提取规律，让系统越用越聪明。比如你总是选"知识干货型"，引擎会记录这个偏好。

**Soul 用户画像**：根据你的使用习惯构建的 AI 人格文档，影响下次推荐卡片的生成方式。使用次数越多，推荐越个性化。

### 用户画像数据存在哪里？可以删除吗？

存储在项目根目录的 `.evolution/` 文件夹中：

```
.evolution/
├── learnings/       # 每次操作的学习记录（JSON）
├── souls/           # 用户画像文件（default.json）
├── patterns/        # 识别出的行为模式
└── rules/           # 晋升为规则的经验
```

**删除方法：** 直接删除 `.evolution/` 文件夹，画像和学习数据会重置为初始状态，不影响系统正常功能。

### 为什么推荐卡片一开始不够准确？

正常现象。Soul 画像需要积累数据才能个性化：

- **刚开始**：使用静态的 3 个默认档案（爆款/知识/品牌）
- **使用 3–5 次后**：AI 开始注入你的历史偏好（平台、目标、风格）
- **使用 10 次以上**：推荐卡片会明显更贴合你的内容风格

### 选择了参数，对 Soul 画像有什么影响？

| 你的操作 | Soul 画像变化 |
|---|---|
| 选推荐卡片（而不是表单） | 系统认为你倾向 AI 辅助，调整为简洁推荐风格 |
| 选参数表单 | 系统认为你偏好精细控制，下次更多细节 |
| 多次选"爆款流量型" | 偏好策略列表更新，风险容忍度提升 |
| 多次选"品牌调性型" | 风险容忍度降低，推荐倾向稳健 |
| 选了 AI 推荐的档案 | AI 信任度提升，推荐卡片更大胆 |
| 选了非推荐档案 | AI 信任度微降，后续推荐更保守 |
